# Product

## Register

product

## Users

Developers evaluating `@pulgueta/wompi` in a realistic TanStack Start application. They need a concise ecommerce flow that shows database reads, authenticated user creation, order persistence, and Wompi checkout creation without a large application architecture.

## Product Purpose

Show how to integrate Wompi payments into a full-stack React app backed by Turso SQLite and Drizzle. Success means a developer can read the code, run the storefront with mock products, sign in, create an order, and see where Wompi payment-link metadata is produced and stored.

## Brand Personality

Practical, confident, and commerce-ready. The interface should feel like a credible boutique storefront while keeping the code easy to inspect.

## Anti-references

Avoid generic SaaS starter pages, overbuilt ecommerce platform architecture, and decorative UI that distracts from the Wompi integration path.

## Design Principles

- Keep the payment path visible from product selection through persisted order metadata.
- Prefer production-like data boundaries over demo-only shortcuts.
- Make missing environment configuration explicit and actionable.
- Use real UI states for signed-out, cart, checkout, and recent orders.

## Accessibility & Inclusion

Aim for WCAG AA contrast, semantic form controls, keyboard-operable actions, and reduced-motion-safe interactions.
